MPFR version 2.1.2 (released on 3 August 2005)

[download] [documentation] [bugs] [changes]

MPFR 2.1.2 requires GMP 4.1.0 or higher. This is the latest version of the 2.1 branch, which is no longer supported (newly discovered bugs will probably never be fixed on this branch). Please upgrade to MPFR 2.2.0 (or higher, when new versions are released).

Download

MPFR 2.1.2 can be downloaded in the following archive formats:

  • mpfr-2.1.2.tar.bz2 (tar/bzip2, 486 KB) (signature),
  • mpfr-2.1.2.tar.gz (tar/gzip, 609 KB) (signature),
  • mpfr-2.1.2.zip (zip, 862 KB) (signature).

The signatures can be checked with Laurent Fousse's key (or grab this key A1696D2B from the keyserver keyserver.noreply.org).

MPFR is also available as the official Debian package libmpfr-dev.

Bugs

Fixed bug: exp(-eps) is somtimes wrong when rounding toward zero (the test is also wrong). This is fixed by the mpfr_exp patch (with updated tests).

In addition to the bugs mentioned in the BUGS file:

  • There are possible bugs with huge precisions (greater than 230) and with exponent ranges that do not allow to represent some numbers, such as 1 or 1/2.
  • In mpfr_get_ld, the rounding direction can be wrong if |x| < LDBL_MIN.

The following bugs are present in MPFR 2.1.2, but are fixed in the 2.2 branch and above:

  • Memory leak in mpfr_sum (more precisely, in the mpfr_count_sort internal function).
  • MPN_COPY may call memcpy with identical source and destination pointers; this is incorrect and may lead to an incorrect behavior with some (rare?) C implementations.
  • On some architectures, unsigned long bit-fields are used (this is a GCC extension and may not be supported by other compilers).
  • Small bugs in mpfr_set_str_binary internal function; they should not have any consequences with the current code.
